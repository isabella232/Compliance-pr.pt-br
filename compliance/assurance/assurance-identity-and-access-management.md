<span data-ttu-id="99d30-109">Engenheiros atribuídos a uma equipe de serviço para dar suporte à qualificação de solicitação de serviços de produção para uma conta de equipe de serviço por meio da Ferramenta de Gerenciamento de Identidade (IDM).</span><span class="sxs-lookup"><span data-stu-id="99d30-109">Engineers assigned to a service team to support production services request eligibility for a service team account through the Identity Management Tool (IDM).</span></span> <span data-ttu-id="99d30-110">A solicitação de qualificação dispara uma série de verificações de funcionários para garantir que o engenheiro tenha passado todos os requisitos de triagem na nuvem, concluído o treinamento necessário e recebido aprovação de gerenciamento apropriada antes da criação da conta.</span><span class="sxs-lookup"><span data-stu-id="99d30-110">The request for eligibility triggers a series of personnel checks to ensure the engineer has passed all cloud screening requirements, completed necessary training, and received appropriate management approval prior to account creation.</span></span> <span data-ttu-id="99d30-111">Somente depois de atender a todos os requisitos de qualificação pode ser criada uma conta de equipe de serviço para o ambiente solicitado.</span><span class="sxs-lookup"><span data-stu-id="99d30-111">Only after meeting all eligibility requirements can a service team account be created for the requested environment.</span></span>

## <a name="how-does-microsoft-365-handle-remote-access-to-production-systems"></a><span data-ttu-id="99d30-117">Como o Microsoft 365 lida com o acesso remoto a sistemas de produção?</span><span class="sxs-lookup"><span data-stu-id="99d30-117">How does Microsoft 365 handle remote access to production systems?</span></span>

<span data-ttu-id="99d30-118">Os componentes do sistema do Microsoft 365 são ativos em datacenters geograficamente separados das equipes de operações.</span><span class="sxs-lookup"><span data-stu-id="99d30-118">Microsoft 365 system components are housed in datacenters geographically separated from the operations teams.</span></span> <span data-ttu-id="99d30-119">O pessoal do datacenter não tem acesso lógico aos sistemas do Microsoft 365.</span><span class="sxs-lookup"><span data-stu-id="99d30-119">Datacenter personnel do not have logical access to Microsoft 365 systems.</span></span> <span data-ttu-id="99d30-120">Como resultado, a equipe de serviço do Microsoft 365 gerencia o ambiente por meio do acesso remoto.</span><span class="sxs-lookup"><span data-stu-id="99d30-120">As a result, Microsoft 365 service team personnel manage the environment through remote access.</span></span> <span data-ttu-id="99d30-121">A equipe de serviço que exige acesso remoto para dar suporte ao Microsoft 365 só tem acesso remoto após a aprovação de um gerente autorizado.</span><span class="sxs-lookup"><span data-stu-id="99d30-121">Service team personnel who require remote access to support Microsoft 365 are only granted remote access after approval from an authorized manager.</span></span> <span data-ttu-id="99d30-122">Todo o acesso remoto usa TLS compatível com FIPS 140-2 para conexões remotas seguras.</span><span class="sxs-lookup"><span data-stu-id="99d30-122">All remote access uses FIPS 140-2 compatible TLS for secure remote connections.</span></span>

<span data-ttu-id="99d30-123">O Microsoft 365 usa Estações de Trabalho de Administrador Seguro (SAWs) para o acesso remoto da equipe de serviço para ajudar a proteger os ambientes do Microsoft 365 contra comprometimento.</span><span class="sxs-lookup"><span data-stu-id="99d30-123">Microsoft 365 uses Secure Admin Workstations (SAWs) for service team remote access to help protect Microsoft 365 environments from compromise.</span></span> <span data-ttu-id="99d30-124">Essas estações de trabalho foram projetadas especificamente para evitar perda intencional ou não intencional de dados de produção, incluindo o bloqueio de portas USB e a limitação do software disponível no SAW para o que é necessário para dar suporte ao ambiente.</span><span class="sxs-lookup"><span data-stu-id="99d30-124">These workstations are specifically designed to prevent intentional or unintentional loss of production data, including locking down USB ports and limiting the software available on the SAW to what is required for supporting the environment.</span></span> <span data-ttu-id="99d30-125">Os SAWs são acompanhados e monitorados de perto para detectar e impedir o comprometimento mal-intencionado ou inadvertido dos dados do cliente pelos engenheiros da Microsoft.</span><span class="sxs-lookup"><span data-stu-id="99d30-125">SAWs are closely tracked and monitored to detect and prevent malicious or inadvertent compromise of customer data by Microsoft engineers.</span></span>
